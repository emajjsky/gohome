# EACP 算法闭环验收记录

日期：2026-07-21

## 验收范围

- 树莓派端人物、姿态、场景和跌倒时序联合判断
- 正常坐下、弯腰、蹲下及沙发/床躺姿的误报抑制
- 正式事件截图与结构化证据上传
- 云端多模态模型复核
- 云端结论、恢复状态、App 待确认消息的状态一致性
- 管理端实时画面、边缘证据和云端复核结果展示

## 技术链路

1. 摄像头共享流持续取帧，预览与推理不重复拉取 RTSP。
2. 人形出现后进入 EACP 连续姿态感知，模型锚点与帧间跟踪协同工作。
3. 跌倒不是由单帧“躺姿”直接触发，而是由姿态、垂直位移、速度、持续时间、场景和轨迹连续性共同形成因子图证据。
4. 沙发、床等正常躺姿区域默认抑制事件，只有当前帧因子图重新确认动态跌倒过程时才允许越过场景抑制。
5. 边缘规则形成正式事件后上传截图和证据，云端多模态模型完成二次复核。
6. 云端复核结果是最终事件结论；人物重新站立只记录恢复状态，不自动确认、归档或覆盖模型结论。

## 已修复风险

- 正常躺姿区域不再继承上一帧的快速下降证据。
- 低位坐姿必须具备近期连续下降信号，不能仅凭低位框触发跌倒。
- 人物触及画面底边时纳入边缘裁切抑制，避免半身框误判。
- 人物恢复站立不再覆盖云端复核结论，也不会自动关闭 App 待确认消息。
- 管理端弱火焰视觉线索不再覆盖人物活动主状态，只作为次级证据显示。

## 验收结论

当前版本已经形成“边缘连续感知 -> 时序因子图 -> 正式事件 -> 云端多模态复核 -> 恢复与用户确认分离 -> 双端日志留存”的可运行闭环。算法能力可以进入管理端展示和后续数据集按需增强阶段，但仍需在最终路演前补录一组无遮挡、全身入镜的正样本作为视觉证据。
